SMPP Protocol Methods (Supplier Core)

The platform splits SMPP functionality into individual, isolated execution submodules. Click any operation to inspect its purpose and local supplier file logic:

Bind Transmitter /app/smpp_methods/bind_transmitter.py
Transmitter
Purpose: Authenticates with the SMSC to establish an outbound-only session.
Application: Used for marketing bulk campaigns and one-way verification codes where incoming traffic is not expected or processed.
Bind Receiver /app/smpp_methods/bind_receiver.py
Receiver
Purpose: Registers the client with the SMSC to read and collect incoming SMS traffic.
Application: Listens for Delivery Receipts (DLR) and user inbound replies (Mobile Originated).
Bind Transceiver /app/smpp_methods/bind_transceiver.py
Transceiver
Purpose: Standard modern login allowing concurrent message transmission and reception over a unified port connection.
Application: Keeps sockets minimal while getting instant callback events.
Submit SM /app/smpp_methods/submit_sm.py
Outbound
Purpose: Primary SMPP PDU payload that carries the individual SMS text, recipient, and requested DLR flags to the SMSC.
Deliver SM /app/smpp_methods/deliver_sm.py
Inbound
Purpose: Incoming PDU containing messages sent by end users back to your virtual/allocated numbers, auto-parsing payout metrics.
Enquire Link /app/smpp_methods/enquire_link.py
Heartbeat
Purpose: Regular polling command (PING) ensuring active session parameters remain established at the gateway, avoiding socket timeout.
Unbind Graceful /app/smpp_methods/unbind.py
Close Session
Purpose: Safely informs the SMSC of intended disconnect, preventing raw TCP socket leakages or authentication lockdowns.
Query SM /app/smpp_methods/query_sm.py
Status Check
Purpose: Active polling method to retrieve current delivery statuses of messages if standard delivery receipts (DLR) failed to push.
Cancel SM /app/smpp_methods/cancel_sm.py
Cancel Job
Purpose: Attempts to halt or abort a message dispatch if it remains pending in the supplier delivery queue.
SMPP over HTTP API /app/smpp_methods/smpp_api.py
HTTP REST
Purpose: Outbound REST proxying to Jasmin HTTP interfaces or Twilio API, securing connection details behind web endpoints.
SMPP Protocol Standard Reference

SMPP (Short Message Peer-to-Peer)

SMPP is an open, industry-standard telecommunications protocol designed to simplify the exchange of short messages (SMS) between External Short Message Entities (ESME) and Short Message Service Centers (SMSC).

Standard PDU Exchange Flow
Client (ESME) Server (SMSC) | | | ---- BIND_TRANSCEIVER PDU ---------> | [Awaiting auth] | <--- BIND_TRANSCEIVER_RESP PDU ----- | [Session bound] | | | ---- SUBMIT_SM (Transmit SMS) -----> | [Enqueuing msg] | <--- SUBMIT_SM_RESP (Get MSG_ID) --- | [Success] | | | <--- DELIVER_SM (Delivery Report) -- | [Inbound event] | ---- DELIVER_SM_RESP --------------> | [Acknowledged]
