SMPP Protocol Reference

SMPP Protocol Overview

SMPP (Short Message Peer-to-Peer) is an open industry standard protocol used for SMS communications between external SMS applications (ESMEs) and SMS centers (SMSCs).

PDU Operations
Bind Operations
  • Bind_Transmitter - Send SMS only
  • Bind_Receiver - Receive SMS only
  • Bind_Transceiver - Send & Receive
Message Operations
  • Submit_SM - Send message to SMSC
  • Submit_SM_Resp - Response with message_id
  • Deliver_SM - Receive message from SMSC
  • Enquire_Link - Keep connection alive
Popular SMPP Providers
  • Jasmin SMS - Open source SMS gateway (GitHub: jookies/jasmin)
  • Kannel - Open source WAP/Push gateway
  • AWS SNS - Amazon Simple Notification Service
  • Twilio - Via SMPP gateway connector
  • NOWSMS - Commercial SMS gateway
Connection Parameters
Parameter Description
system_id ESME identifier for authentication (like username)
password Authentication password
system_type Optional session type identifier
interface_version SMPP version (default: 3.4 or 5.0)
enquire_link Heartbeat PDU to maintain connection (usually every 10-60 sec)
Note: Jasmin SMS installation requires a Linux server with Python and Redis. Visit https://github.com/jookies/jasmin for installation guide.
